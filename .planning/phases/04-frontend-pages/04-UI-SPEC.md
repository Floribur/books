## Spacing Scale

Declared values (must be multiples of 4). **Use CSS custom properties — never raw px values in new CSS files.**

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, tag inline padding, skeleton gaps |
| `--space-sm` | 8px | Metadata separator spacing, list-row inner padding |
| `--space-md` | 16px | Default element spacing, page horizontal padding |
| `--space-lg` | 24px | Section heading bottom margin, detail column gap |
| `--space-xl` | 32px | Page vertical padding, stats strip gap |
| `--space-2xl` | 48px | Major section breaks |
| `--space-3xl` | 64px | Page-level spacing (unused in Phase 4) |

Exceptions:
- Touch targets (arrow buttons, tag links): minimum 44px height, per established pattern in `BookGrid.css` and `Toast.css`.
- Mobile floating cover width: **96px** (fixed, not a spacing token) — smallest usable cover beside readable text.

## Typography

All values from `frontend/src/styles/tokens.css` and `typography.css`. **Do not introduce new type sizes in Phase 4.**

| Role | Token | Size | Weight | Line Height | Font | Usage in Phase 4 |
|------|-------|------|--------|-------------|------|-------------------|
| Display | `--text-display` | 36px | 700 | 1.1 | Playfair Display | Page-level headings (unused in Phase 4 — use Heading instead) |
| Heading | `--text-heading` | 24px | 700 | 1.2 | Playfair Display | Page section headings (author name, genre name, "Authors", "Genres", "Reading Challenge") |
| Body | `--text-body` | 16px | 400 | 1.6 | Inter | Book description, metadata body text |
| Label | `--text-label` | 13px | 400 | 1.4 | Inter | Genre tags, stats labels, metadata pill (year · pages · date), list row secondary text, "Show more/less" link, footnote |

Weight used: **400** (body/label) and **700** (all headings). No other weights.

Emphasis exception: book title on detail page uses `font-weight: 700` at `--text-heading` size with Playfair Display — matching the `h2` rule already established in `typography.css`.

## Component Inventory — New for Phase 4

### 1. BookDetailPage

**Route:** `/books/:slug`

**Desktop layout (≥768px):**
```
[Cover column 200px fixed] | [Metadata column, fills rest]
```
- Outer grid: `display: grid; grid-template-columns: 200px 1fr; gap: var(--space-xl);`
- Cover: `BookCover` component reused as-is, with `loading="eager"` (above the fold).
- Metadata column order (top to bottom):
  1. Book title — `h1` at `--text-heading`/700/Playfair Display/`--color-primary`
  2. Author link(s) — `--text-body`/400/Inter/`--color-accent`, comma-separated `<a>` tags linking to `/authors/:slug`
  3. Genre tags — inline flex row, wrapping, gap `--space-xs`
  4. Metadata line — `--text-label`/400/Inter/`--color-text-muted`: "Published {year} · {N} pages · Read {date}" (omit fields with null values; use `·` U+00B7 as separator)
  5. Description — `--text-body`/400, expandable (see below)

**Mobile layout (<768px):**
- Cover floats left: `float: left; width: 96px; margin: 0 var(--space-md) var(--space-sm) 0;`
- Metadata renders as normal block flow — text wraps around the float.
- A clearfix `<div style="clear: both">` appears after the metadata column to contain the float.
- No stacking or full-width cover on mobile. Cover remains 96px wide.

**Genre tags (pill style):**
```css
.genre-tag {
  display: inline-block;
  padding: var(--space-xs) var(--space-sm);
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: 4px;
  font-size: var(--text-label);
  color: var(--color-text);
  text-decoration: none;
  line-height: var(--leading-label);
  min-height: 28px;        /* comfortable without 44px requirement — passive link */
  transition: background 0.15s ease;
}
.genre-tag:hover {
  background: var(--color-border);
}
```

**Description expand/collapse (D-04):**
- Threshold: >640 characters OR >8 rendered lines. Check character count (`description.length > 640`) — simpler and deterministic.
- Collapsed state: `overflow: hidden; display: -webkit-box; -webkit-line-clamp: 8; -webkit-box-orient: vertical;`
- "Show more" link: renders below clamped text as a block-level `<button>` styled as a link. Text: **"Show more"**. Color: `--color-accent`. Font: `--text-label`. No underline default, underline on hover.
- Expanded state: `overflow: visible; display: block;` (remove clamp). "Show less" link appears below full text. Text: **"Show less"**.
- Transition: `max-height` transition is **not used** due to indeterminate content height. Instead, toggle CSS class `description--expanded` on the wrapper — instant visibility change. `prefers-reduced-motion` is satisfied automatically (no animation).
- Accessibility: button has `aria-expanded={isExpanded}` attribute.

**Loading state:** Skeleton of the two-column layout:
- Cover column: single `SkeletonCard`-style block (aspect-ratio 2/3, 200px wide, `--color-skeleton` background).
- Metadata column: 4 skeleton text lines (widths: 70%, 40%, 60%, 90%) with `--space-sm` gap. Rendered as `<div>` elements with `height: 14px; border-radius: 4px; background: var(--color-skeleton)`.

**Error state:** Reuse existing `<Toast>` component. Message: **"Couldn't load book details. Try refreshing the page."**

### 2. AuthorsPage (Author Index)

**Route:** `/authors`

**Layout:** Single column, full width within `app-content`. Heading "Authors" (`h1` at `--text-heading` size).

**List design:**
- `<ol>` or `<ul>` with `list-style: none`.
- Each row is an `<a>` wrapping the full row width: author name (left) and book count (right), separated by flex spacer.
- Row height: min 44px (touch target).
- Row padding: `var(--space-sm) var(--space-md)`.
- Author name: `--text-body`/400/Inter/`--color-text`.
- Book count: `--text-label`/400/Inter/`--color-text-muted` — format: **"{N} books"** (singular: "1 book").
- Row hover: `background: var(--color-surface)`.
- Row separator: `border-bottom: 1px solid var(--color-border)` on each row.
- First row: `border-top: 1px solid var(--color-border)`.
- No A-Z jump anchors (D-12).
- Sort: alphabetical by surname (backend responsibility; frontend renders as received).

**Loading state:** 8 skeleton rows — each row is a `--color-skeleton` block at `height: 20px`, full width, with `border-radius: 4px`.

**Empty state:** Should not occur (there are always authors). If it does: heading remains, body text: **"No authors found."**

**Error state:** Toast — **"Couldn't load authors. Try refreshing the page."**

### 3. AuthorDetailPage

**Route:** `/authors/:slug`

**Layout:**
- Author name as `h1` at `--text-heading`/700/Playfair Display/`--color-primary`.
- Optional subheading: **"{N} books"** in `--text-label`/400/Inter/`--color-text-muted` immediately below the name.
- `BookGrid` component reused below the heading, filtered to this author.

**BookGrid adaptation:** Pass a custom `queryKey` of `['books', 'author', slug]` and a `fetchFn` of `fetchBooksByAuthor(slug, cursor)`. BookGrid handles infinite scroll and skeletons internally.

**Loading state (heading area):** Single skeleton line (60% width, 28px height) for the author name.

**Empty state:** Should not occur. If it does: BookGrid renders its own empty state (see below).

**Error state (author fetch):** Toast — **"Couldn't load author. Try refreshing the page."**

### 4. GenresPage (Genre Index)

**Route:** `/genres`

**Layout:** Single column. Heading "Genres" (`h1` at `--text-heading`).

**Bar visualization (D-05, D-06, D-07):**
- Each genre is a row rendered as `<a>` (entire row is a link to `/genres/:slug`).
- Row: `display: flex; align-items: center; gap: var(--space-md); padding: var(--space-sm) 0; border-bottom: 1px solid var(--color-border); min-height: 44px;`
- Genre name: fixed left column, width `160px`, `--text-body`/400/`--color-text`, no-wrap overflow ellipsis.
- Bar container: `flex: 1; height: 16px; background: var(--color-surface); border-radius: 4px; overflow: hidden;`
- Bar fill: `height: 100%; border-radius: 4px; background: rgba(109, 35, 62, 0.3);` Width set via inline style: `width: calc(count / max * 100%)` — pass `maxCount` as prop, compute per-row.
- Count: right column, `--text-label`/400/`--color-text-muted`, `min-width: 48px; text-align: right;` — format: **"{N}"** (number only; the "books" label is implicit).
- Row hover: bar fill darkens slightly — `background: rgba(109, 35, 62, 0.45)`.

```css
/* GenresPage.css — derived token (local only) */
:root {
  --color-genre-bar: rgba(109, 35, 62, 0.3);
  --color-genre-bar-hover: rgba(109, 35, 62, 0.45);
}
```

**Loading state:** 6 skeleton rows — each row: 3 skeleton blocks (160px label placeholder, flex-1 bar placeholder, 48px count placeholder) at appropriate heights.

**Empty state:** Should not occur. If it does: **"No genres found."**

**Error state:** Toast — **"Couldn't load genres. Try refreshing the page."**

### 6. ReadingChallengePage

**Route:** `/reading-challenge`

**Layout (top to bottom):**
1. Page heading: **"Reading Challenge"** — `h1` at `--text-heading`/700/Playfair Display/`--color-primary`.
2. Year selector strip (see below).
3. Stats strip (see below).
4. `BookGrid` filtered to selected year.

---

#### Year Selector Component

**Anatomy:** `[← prev] [2024] [next →]` — centered horizontal strip.

- Outer: `display: flex; align-items: center; justify-content: center; gap: var(--space-md);`
- Prev/next: `<button>` with SVG chevron icon (inline, 20×20). Width/height: 44px (touch target). Background: none. Border: none. Color: `--color-text-muted`. Hover: `--color-accent`. Disabled (first/last year): `opacity: 0.3; cursor: not-allowed`.
- Year display: `--text-heading`/700/Playfair Display/`--color-primary`, `min-width: 80px; text-align: center;`
- Keyboard: `onKeyDown` on the prev button — ArrowLeft navigates prev; ArrowRight navigates next. (Or attach to the container and delegate.)
- URL: `?year={YYYY}` — use `useSearchParams` from React Router. On mount: if no param, default to most recent year with books (D-08); if param exists, use it.
- Disabled state: prev button disabled when at oldest year; next button disabled when at most recent year.

---

#### Stats Strip

**Layout:** Horizontal flex row, centered, wraps on narrow screens.
- Outer: `display: flex; gap: var(--space-xl); flex-wrap: wrap; justify-content: center; padding: var(--space-md) var(--space-xl); background: var(--color-surface); border-radius: 8px; margin: var(--space-md) 0;`

Each stat:
- `display: flex; flex-direction: column; align-items: center; gap: var(--space-xs);`
- Stat value: `--text-heading`/700/Playfair Display/`--color-primary`
- Stat label: `--text-label`/400/Inter/`--color-text-muted`

Three stats (D-09):
1. **Books Read** — value: `{N}`, label: `"books read in {year}"`
2. **Pages Read*** — value: `{N}` (sum of `page_count`), label: `"pages read*"`. Hidden if sum is 0 (no page data for year).
3. **Longest Book** — value: book title (truncated at 24 chars with `…` if needed), label: `"{N} pages"`. Hidden if no page data for year.

**Footnote (D-10):** Below the stats strip, one `<p>` in `--text-label`/400/`--color-text-muted`:
**"* Based on available page data (not all books have page counts)."**
Only render the footnote when stats 2 or 3 are visible.

**Loading state (stats strip):** Three skeleton blocks side by side — each `80px × 48px`, `--color-skeleton`, `border-radius: 4px`.

---

**BookGrid for Reading Challenge:**
- `queryKey: ['books', 'year', String(selectedYear)]`
- `fetchFn: fetchBooksByYear(selectedYear, cursor)`
- Same BookGrid component; reads year from state (not URL directly — ReadingChallengePage owns the year state and passes it as a prop/closure).

**Empty state (year with no books):** BookGrid renders an empty grid. ReadingChallengePage adds a message above the (empty) grid:
- Heading: (none — page heading already shown)
- Body: **"No books recorded for {year}."**
- Rendered as `<p>` in `--text-body`/400/`--color-text-muted`, centered, `padding: var(--space-2xl) 0`.
- Per D-08, this should not occur for past years with data.

**Error state (years list fetch):** Toast — **"Couldn't load years. Try refreshing the page."**
**Error state (books fetch):** Handled by BookGrid's own Toast.

## Interaction Contracts

### Global patterns (from Phase 3 — reuse unchanged)
- Focus ring: `outline: 2px solid var(--color-accent); outline-offset: 2px` on `:focus-visible` — already global.
- Hover transitions: `transition: 0.15s ease` — match all interactive elements in Phase 4.
- Toast positioning: `position: fixed; top: 16px; right: 16px` — match existing Toast.css.
- Touch targets: minimum 44px height for all buttons and interactive rows.
- `prefers-reduced-motion`: no Phase 4 component introduces motion beyond the description class-toggle (which has none).

### New interaction contracts

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Genre row hover | `mouseenter` on `<a>` row | Bar fill changes from `rgba(109,35,62,0.3)` → `rgba(109,35,62,0.45)` |
| Genre row active | `mousedown` | Standard browser `color` shift (no custom active state) |
| Author row hover | `mouseenter` on `<a>` row | Row `background: var(--color-surface)` |
| "Show more" click | `click` on button | `isExpanded` state → `true`; description class `description--expanded` added; button text → "Show less" |
| "Show less" click | `click` on button | `isExpanded` → `false`; class removed; button text → "Show more" |
| Year prev arrow | `click` or `ArrowLeft` key | Decrement `selectedYear`; update `?year=` URL param |
| Year next arrow | `click` or `ArrowRight` key | Increment `selectedYear`; update `?year=` URL param |
| Year arrow disabled | At first/last year | `disabled` attribute; `opacity: 0.3`; no keyboard response |
| Book detail cover | On `<img>` error | Falls back to `BookCover`'s existing `placeholderUrl` function — no change needed |

## CSS Architecture

Follow the Phase 3 pattern exactly: **one `.css` file per page component**, co-located in `pages/` or `components/`:

```
pages/
  BookDetailPage.tsx
  BookDetailPage.css
  AuthorsPage.tsx
  AuthorsPage.css
  AuthorDetailPage.tsx
  AuthorDetailPage.css
  GenresPage.tsx
  GenresPage.css
  GenreDetailPage.tsx
  GenreDetailPage.css
  ReadingChallengePage.tsx
  ReadingChallengePage.css
components/
  YearSelector.tsx
  YearSelector.css
  StatsStrip.tsx
  StatsStrip.css
  DescriptionBlock.tsx      (manages Show more/less state)
  DescriptionBlock.css
```

Rules:
- Use `var(--space-*)`, `var(--color-*)`, `var(--text-*)`, `var(--font-*)`, `var(--leading-*)` throughout.
- No hardcoded color values except `rgba(109, 35, 62, 0.3)` / `rgba(109, 35, 62, 0.45)` in `GenresPage.css` (genre bar — derived from brand primary, documented above).
- No inline styles except: (a) genre bar `width` (computed value), (b) mobile cover `float`.
- Class names follow BEM-lite: `block`, `block-element`, `block--modifier`.
