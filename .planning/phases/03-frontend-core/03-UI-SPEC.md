## Spacing Scale

All tokens are CSS custom properties (`--space-{token}`). Multiples of 4 only.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, badge padding, inline gaps between author names |
| `--space-sm` | 8px | Card internal padding, input padding, nav link gaps |
| `--space-md` | 16px | Default element spacing, card gutters, sidebar link vertical padding |
| `--space-lg` | 24px | Section internal padding, bio section gap between photo and text |
| `--space-xl` | 32px | Gap between home page sections (Now Reading ↔ Books Read) |
| `--space-2xl` | 48px | Top-of-page vertical breathing room, sidebar top padding |
| `--space-3xl` | 64px | Major page-level vertical rhythm between hero and content |

**Exceptions:**
- Touch targets (mobile nav hamburger, dark mode toggle): minimum **44px × 44px** regardless of visible icon size — achieved via padding, not size inflation
- Book cover `aspect-ratio: 2/3` — not governed by spacing scale; intrinsic ratio constraint

## Typography

Two fonts, four type roles, two weights only.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Inter | 16px | 400 (regular) | 1.6 | Bio text, book descriptions, general prose |
| Label | Inter | 13px | 400 (regular) | 1.4 | Book card titles, author names on cards, nav link text, section meta |
| Heading | Playfair Display | 24px | 700 (bold) | 1.2 | "Now Reading", "Books Read" section headings, bio person name |
| Display | Playfair Display | 36px | 700 (bold) | 1.1 | Page-level heading ("Flo's Library"), reserved for site identity only |

**Weight declaration:** Exactly 2 — regular (400) for all body/label roles; bold (700) for all heading/display roles.

**CSS variables:**
```css
--font-body: 'Inter', system-ui, sans-serif;
--font-heading: 'Playfair Display', Georgia, serif;
--text-body: 16px;
--text-label: 13px;
--text-heading: 24px;
--text-display: 36px;
--leading-body: 1.6;
--leading-label: 1.4;
--leading-heading: 1.2;
--leading-display: 1.1;
```

## Component Contracts

### BookCover
- `aspect-ratio: 2/3` (portrait, locked)
- `object-fit: cover`
- `loading="lazy"` on all images below the fold
- `loading="eager"` on Now Reading covers (above fold)
- Placeholder: CSS gradient `var(--color-skeleton)` fills the 2:3 rectangle while image loads; no broken-image icon (UI-09)
- Alt text: `"{title} cover"` — always set

### BookCard (Books Read)
- Cover + title (truncate at 2 lines) + author name(s) (truncate at 1 line)
- No read date, no genres, no year on card (D-08)
- Entire card is a `<a>` (cover and title both navigate — HOME-04)
- Hover: subtle `transform: translateY(-2px)` + `box-shadow` lift — no color change
- Card min-width: governed by grid, no explicit min-width override

### NowReadingCard (Now Reading)
- Larger cover: same `aspect-ratio: 2/3`, but wider column = physically larger render
- Shows: cover + title + author (D-07)
- Visually differentiated from BookCard: `font-size` for title bumped from label (13px) to 16px body
- No cap on title truncation (single card has more room)

### BookGrid (Books Read)

Column counts at breakpoints (D-10):

| Breakpoint | Columns | Notes |
|------------|---------|-------|
| ≥1400px (wide desktop) | 6 | Dense grid — covers are small, many visible |
| 1024–1399px (desktop) | 5 | Standard desktop view |
| 768–1023px (tablet) | 4 | Comfortable tablet |
| <768px (mobile) | 2 | Per UI-10 — locked requirement |

CSS: `grid-template-columns: repeat({n}, 1fr)` at each breakpoint.

### NowReadingGrid

Column counts (D-06: fewer columns, larger/more prominent than Books Read):

| Breakpoint | Columns | Notes |
|------------|---------|-------|
| ≥1024px (desktop) | 4 | Cap at D-05 — 4 books maximum displayed |
| 768–1023px (tablet) | 3 | — |
| <768px (mobile) | 2 | Matches Books Read mobile columns |

**Now Reading cap: 4 books** (agent discretion resolved — 4 fits naturally at 1024px widths). Books beyond 4 are silently omitted (D-05).

**Section hidden entirely** when API returns empty array (D-04) — no empty state rendered.

### Sidebar
- Width: 240px (desktop, fixed left)
- Contains: site logo/title, nav links (Home, Authors, Genres, Reading Challenge), dark/light toggle (near bottom of nav, D-11)
- Nav links: Inter 13px, weight 400; active link uses `--color-primary`; left border accent `3px solid var(--color-accent)` on active link only
- Mobile: collapses — see Mobile Navigation below

### Dark/Light Toggle (D-12, D-13)
- Sun icon (light mode active) / Moon icon (dark mode active) — from Lucide React
- Icon size: 20px visual, 44×44px touch target via padding
- No label text (D-12); `aria-label="Switch to dark mode"` / `"Switch to light mode"` for accessibility
- Icon fill color: `var(--color-accent)` when active theme is engaged
- Behavior: reads `prefers-color-scheme` on first load → persists in `localStorage` key `"theme"` → toggles `[data-theme]` on `<html>` on click (D-13)

### Skeleton Cards (D-14)
- Match BookCard dimensions exactly (same grid slot)
- Cover placeholder: CSS gradient rectangle using `--color-skeleton`
- Two text line placeholders below: 80% width (title) + 60% width (author), height 12px, border-radius 4px, same skeleton gradient
- No animation required for v1 (shimmer is a v2 enhancement)
- Render 12 skeleton cards while initial Books Read data loads

### Error Toast (D-15)
- CSS-only — no external toast library
- Fixed position: top-right, `top: 16px; right: 16px`
- Background: `var(--color-destructive)`, white text
- Padding: 12px 16px; border-radius: 6px
- Auto-dismiss: 5 seconds via CSS animation + `setTimeout` remove
- Close button (×) — 44×44px touch target — dismisses immediately
- Stacks vertically if multiple errors (gap 8px)
- `role="alert"` for screen readers

### Bio Section (D-01, D-02, D-03)
- Content source: `src/content/bio.md` — raw import parsed with `marked` library (no Vite plugin required)
- Desktop: photo left (240px wide, 2:3 ratio, `border-radius: 4px`), text right — standard author bio layout (D-02)
- Mobile: photo first (full-width, max 280px centered), text below (D-16 agent discretion — logical reading order)
- Goodreads profile link in bio text: `https://www.goodreads.com/user/show/79499864-florian` — styled with `--color-accent` underline (D-03)
- Bio gap at desktop: `var(--space-lg)` between photo and text columns

### Mobile Navigation (D-16)
- Hamburger button in sticky top bar (height 48px, `--color-surface` background, `--color-primary` hamburger icon)
- Hamburger opens sidebar as left-side drawer overlay (slides in from left, `transition: transform 0.25s ease`)
- Semi-transparent backdrop (`rgba(0,0,0,0.4)`) covers main content — click backdrop to close
- Focus trap while open (Tab cycles through nav links + close button)
- Drawer closes on nav link click (navigating away)
- `aria-expanded` on hamburger button reflects open state
- Breakpoint: drawer behavior activates below 768px

## Infinite Scroll Contract (UI-01, UI-02, UI-03)

- **First load:** 24 books fetched (page size = 24)
- **Trigger:** Intersection Observer sentinel `div` at bottom of grid — `threshold: 0.1`
- **"Load More Books" button:** Visible below grid, always rendered as accessible fallback (UI-02); same `fetchNextPage` call as observer
- **Button state:** `disabled` + "Loading…" text while `isFetchingNextPage === true`
- **No more data:** Button hidden when `hasNextPage === false`; sentinel ignored
- **Back-button restoration:** Scroll position saved to `sessionStorage` on card click; on mount, restore via `element.scrollIntoView({ behavior: 'instant' })` if target book ID in URL param (UI-03)

## Accessibility Contract

- All interactive elements: minimum 44×44px touch target
- Color contrast: `--color-primary` `#6d233e` on `--color-background` `#faf8f5` = 7.2:1 ✓ (WCAG AAA)
- Color contrast: `--color-text` `#2c1a1f` on `--color-background` = 12:1 ✓
- Color contrast: `--color-accent` `#c4843a` on `--color-background` = 3.1:1 — accent used for decoration only, never as the sole text color differentiator
- Focus rings: `outline: 2px solid var(--color-accent); outline-offset: 2px` on all interactive elements
- `prefers-reduced-motion`: sidebar drawer transition `transition: none` when reduced motion; no other animations in Phase 3 (sidebar book animation is Phase 5)
- Semantic HTML: `<nav>`, `<main>`, `<section>`, `<article>` (book cards) used correctly
- `<img>` alt text: always set (never empty for cover images)
- Error toast: `role="alert"` for automatic screen reader announcement

## Agent Discretion — Resolved Choices

The following items were explicitly delegated to the agent in CONTEXT.md (§Agent's Discretion). All are resolved here:

| Item | Decision | Rationale |
|------|----------|-----------|
| Now Reading cap (3 or 4) | **4 books** | 4 fills a 4-column desktop grid cleanly; 3 leaves odd whitespace at typical 1024px |
| Books Read grid columns | 6 / 5 / 4 / 2 (desktop wide / desktop / tablet / mobile) | Balanced density — large collection visible at once, mobile stays 2 per UI-10 |
| Now Reading grid columns | 4 / 3 / 2 | Fewer, larger columns than Books Read — emphasises "currently reading" status |
| Mobile nav pattern | Hamburger → left drawer overlay | Universal pattern; tabs would waste vertical space for 4 nav items |
| Toast implementation | CSS-only + `role="alert"` | Zero dependencies; Phase 3 only has one error state per section |
| Bio mobile stack order | Photo first, then text | Establishes visual identity before prose; mirrors editorial book bio convention |
| Markdown parsing | Raw import + `marked` | No Vite plugin needed; marked is 6KB min+gz; zero config |
