## Decisions Made
- `unmarshalRefs[T]` uses `json.Marshal` on `interface{}` then `json.Unmarshal` into slice, because pgx returns `json_agg` results as `[]interface{}` (not `json.RawMessage`)
- `GetCurrentlyReading` route registered before `GetBookBySlug/{slug}` — critical ordering for chi router to avoid matching "currently-reading" as a slug parameter

## Deviations from Plan

**1. unmarshalRefs signature adapted for interface{} instead of json.RawMessage**
- **Found during:** Task 2 (after reading sqlc-generated types)
- **Issue:** Plan showed `unmarshalRefs[T any](raw json.RawMessage)` but generated `Authors`/`Genres` fields are `interface{}` not `json.RawMessage` (pgx scans JSON aggregates as Go values)
- **Fix:** Signature changed to `unmarshalRefs[T any](raw interface{})`, added `json.Marshal(raw)` step before unmarshal
- **Verification:** All 10 tests pass; unmarshalRefs correctly handles nil, empty slice, and populated data
