## Color

All color values must reference CSS custom properties. **No hardcoded hex in new CSS rules.**

### Light Theme

| Role | CSS Var | Hex | Usage |
|------|---------|-----|-------|
| Dominant (60%) | `--color-background` | `#faf8f5` | Page background, sidebar background |
| Secondary (30%) | `--color-surface` | `#f0ebe3` | Cards, nav hover state, drawer backdrop |
| Primary | `--color-primary` | `#6d233e` | Lottie fill color (light mode), favicon SVG fill, headings |
| Accent (10%) | `--color-accent` | `#c4843a` | Links, active nav state, hover states, tags/badges |

| Border | `--color-border` | `#e2d9d0` | Dividers, card borders |
| Destructive | `--color-destructive` | `#b91c1c` | Not used in Phase 5 |

### Dark Theme

| Role | CSS Var | Hex | Usage |
|------|---------|-----|-------|
| Dominant (60%) | `--color-background` | `#15100f` | Page background, sidebar background |
| Secondary (30%) | `--color-surface` | `#221618` | Cards, nav hover state |
| Primary | `--color-primary` | `#e8c4cf` | Lottie fill color (dark mode) — see note below |
| Accent (10%) | `--color-accent` | `#d4943a` | Links, active nav state, tags/badges |
| Text | `--color-text` | `#f0e6e9` | Body text |
| Text muted | `--color-text-muted` | `#9e8085` | Secondary labels |
| Border | `--color-border` | `#3a2428` | Dividers |

**Accent reserved for:** links, active nav state indicator, hover state backgrounds/underlines, genre/tag badges. Never use accent for decorative fills or illustration elements.

**Lottie fill color decision (agent's discretion, resolved here):**
- Light mode sidebar (`--color-background: #faf8f5`): Lottie single fill = `#6d233e` (`--color-primary`). Contrast ratio ~9.1:1 on warm white — WCAG AAA.
- Dark mode sidebar (`--color-background: #15100f`): Lottie single fill = `#e8c4cf` (`--color-primary` dark override). Contrast ratio ~9.4:1 on near-black — WCAG AAA.
- Implementation: post-process downloaded Lottie JSON to replace all color arrays with the target hex (normalized 0–1 RGB). Apply two JSON variants OR use a `useTheme()` hook to switch `animationData` import at runtime.

**Source:** `frontend/src/styles/tokens.css`, `frontend/src/styles/themes.css` — pre-populated.

## Lottie Animation Contract

### Placement
- **Desktop sidebar:** `sidebar-header` div → `<LottieAnimation>` replaces `<span className="sidebar-title">`.
- **Mobile topbar:** unchanged — keeps `<Link to="/" className="sidebar-title">Flo's Library</Link>` as text.
- **Mobile drawer:** `navContent` is shared with desktop sidebar (same JSX). The drawer renders `<LottieAnimation>` at the drawer header. Since the drawer opens via explicit interaction, the animation may play normally (user just interacted). Still respects `prefers-reduced-motion`.

### Sizing
```css
.lottie-animation-wrapper {
  width: 100%;
  max-width: 240px;      /* sidebar width */
  height: auto;          /* proportional to animation native ratio */
  display: block;
}
```

### prefers-reduced-motion
```css
@media (prefers-reduced-motion: reduce) {
  .lottie-animation-wrapper {
    /* Lottie is paused via JS; wrapper visible at frame 0 */
  }
}
```
JS implementation in `LottieAnimation.tsx`:
```ts
const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
// On mount: if prefersReduced → lottieRef.current?.goToAndStop(0, true)
// Otherwise: lottieRef.current?.play()
```

### Accessibility
- Lottie wrapper must have `role="img"` and `aria-label="Animated reading figure — Flo's Library logo"`.
- When `prefers-reduced-motion` is active, `aria-label` remains; static frame is visually equivalent.

### Recoloring
- **Method:** Post-process downloaded Lottie JSON file before committing to `frontend/src/assets/`.
- Replace all `"k"` color arrays in the JSON (format: `[R, G, B, A]` in 0–1 space) with brand colors.
- Light variant JSON: single fill `[0.427, 0.137, 0.243, 1]` (`#6d233e`).
- Dark variant JSON: single fill `[0.910, 0.769, 0.812, 1]` (`#e8c4cf`).
- Import both files; switch based on `useTheme()` hook or `data-theme` attribute detection.

## Page Title Contract

| Page | `document.title` value |
|------|------------------------|
| Home (`/`) | `Flo's Library` |
| Book detail (`/books/:slug`) | `Flo's Library — {book.title}` |
| Authors index (`/authors`) | `Flo's Library — Authors` |
| Author detail (`/authors/:slug`) | `Flo's Library — {author.name}` |
| Genres index (`/genres`) | `Flo's Library — Genres` |
| Genre detail (`/genres/:slug`) | `Flo's Library — {genre.name}` |
| Reading Challenge (`/reading-challenge`) | `Flo's Library — Reading Challenge` |
| 404 / Not Found | `Flo's Library — Page Not Found` |

**Implementation:** Shared `usePageTitle(page?: string)` hook in `frontend/src/hooks/usePageTitle.ts`.

```ts
// Signature
function usePageTitle(page?: string): void
// Effect
document.title = page ? `Flo's Library — ${page}` : "Flo's Library";
```

No external library (React Helmet / react-helmet-async) needed. Plain `useEffect` + `document.title` is sufficient for this non-SEO SPA (SEO handled by Go server Open Graph injection).

## Interaction States
